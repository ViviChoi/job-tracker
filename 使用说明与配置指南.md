# Job Tracker 使用说明与配置指南

## 一、快速开始

1. 将 Job Tracker 文件夹放到任意位置（移动后路径自动适配）
2. 启动配置界面：
   - **Mac 用户**：双击 **`Double Click to Start_Mac.command`**
   - **Windows 用户**：双击 **`Double Click to Start_Windows.bat`**
3. 终端窗口会自动出现并显示启动进度：
   - 首次运行会检测 Python 环境并安装依赖（约1-2分钟，只需一次）
   - 看到 `等待服务就绪.......✅` 后浏览器自动打开
   - 终端窗口保持开着显示运行日志，**按 Ctrl+C 可停止服务**
4. 在配置页面 `http://localhost:8080` 按说明填写配置，点击**保存配置**
5. 点击**启动 Job Tracker**，程序开始在后台运行

> macOS 首次运行 `.command` 文件时若提示"无法打开"，右键 → 打开 → 打开 即可。
> 所有数据（数据库、日志、匹配记录）保存在程序文件夹的 `data/` 子目录中，移动文件夹后自动跟随。
> 需要 **Python 3.10 或更高版本**（由 `python-jobspy` 库决定）。启动脚本会自动检测，未安装时会尝试通过 Homebrew 自动安装并给出手动安装指引。

## 二、配置说明

### 搜索设置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 位置获取方式 | 自动（根据IP）或手动填写城市名 | `Hong Kong` |
| 搜索半径 | 以城市为中心的最大距离（公里） | `30` |
| 只看最近N小时 | 过滤掉旧职位，建议24小时 | `24` |
| 职位关键词 | 支持多个，OR逻辑搜索，回车添加 | `Python Engineer` |
| 关键词别名 | 每个关键词下可手动添加同义词（也可开启 AI 自动学习） | `Developer`, `SWE` |
| 屏蔽公司 | 不想看到的公司，回车添加 | `某外包公司` |
| 屏蔽关键词 | 标题含这些词则过滤，回车添加 | `intern` |
| 抓取间隔 | 多少分钟抓一次，**建议≥30分钟** | `60` |

⚠️ 抓取间隔低于30分钟会触发 LinkedIn 限制风险提示，请谨慎设置。

### 匹配设置

**模式一：关键词打分**
- 需要先在「简历上传」Tab 上传你的简历（.txt / .pdf / .docx）
- 系统从简历中提取技能词，与职位描述对比打分（0-100分）
- 设置最低分数阈值，达到才推送（建议60-75）
- 适合：求职方向固定、简历技能明确

**模式二：AI 判断**
- 用一段自然语言描述你的求职要求，例如：
  > 我做了3年前端开发，想转全栈，优先远程职位，不考虑外包公司，薪资15k以下不看
- AI 直接判断每个职位是否推送，附带一句话理由
- 适合：跨领域求职、转行、要求复杂难以用关键词描述

**AI Provider 选择**

| Provider | 获取地址 | 推荐模型 |
|----------|---------|---------|
| Claude | console.anthropic.com | claude-haiku-4-5-20251001 |
| OpenAI | platform.openai.com | gpt-4o-mini |
| DeepSeek | platform.deepseek.com | deepseek-chat（最便宜） |

### 别名学习设置（可选）

开启后，程序会使用 AI 自动为每个关键词学习不同语言和行业中的同义词、常见别名，扩大 LinkedIn 搜索覆盖面。

| 配置项 | 说明 |
|--------|------|
| 启用别名学习 | 开关，关闭则只用手动别名 |
| AI Provider / API Key | 用于学习调用（与匹配模块共用 provider 即可） |
| 缓存刷新天数 | 每隔多少天重新向 AI 学习一次（默认7天） |

**语言覆盖逻辑**：
- 英文**永远**学习（默认语言）
- 根据你的求职地区自动追加对应语言（如香港 → 中英双语，德国 → 英德双语）
- 如果抓取结果中出现其他语言的 JD，自动后台补学该语言

**缓存机制**：学习结果存储在本地 `data/alias_cache.json`，每次搜索直接读缓存，不重复调用 AI，Token 消耗极低。

你也可以在**别名学习 Tab** 中：
- 手动为每个关键词添加/删除别名
- 点击「AI 建议」获取即时别名推荐
- 点击「立即学习」手动触发一次全量学习
- 查看当前缓存中所有已学到的别名

---

### 存储设置

- **本地数据库路径**：职位数据保存位置，默认 `data/jobs.db`（相对程序目录，移动文件夹后自动跟随）
- **Google Sheets 同步（可选）**：启用后每条职位同时备份到云端
  - 需要 Google Cloud Service Account 凭证文件，详见下方附录
- **简历生成器接口（可选）**：开启后匹配到职位时实时输出给外部程序
  - 文件模式：输出 JSON 文件到指定目录
  - Webhook 模式：POST 到指定 URL

---

### 通知设置

**推送时机**
- 实时推送：匹配到职位立即通知
- 定时推送：设置多个时间点，到时间统一推送（如 09:00、13:00、18:00）

**免打扰时段（可选）**
- 设置开始和结束时间，期间不推送
- 时区自动根据IP识别，跨时区找工作可手动指定（如 `America/New_York`）
- 开启"结束后统一推送"可避免错过免打扰期间的职位

**Telegram 配置**
1. 在 Telegram 搜索 `@BotFather`
2. 发送 `/newbot`，按提示创建，获得 Bot Token
3. 和你的 Bot 发一条消息
4. 访问 `https://api.telegram.org/bot<你的Token>/getUpdates` 获取 Chat ID
5. 填入配置页

**Email 配置（可选）**
- Gmail 需要开启两步验证后生成「应用专用密码」
- SMTP 服务器填 `smtp.gmail.com`，端口 `587`

---

### 简历上传

- 支持格式：`.txt`、`.pdf`、`.docx`
- 点击上传区域选择文件，或直接拖拽
- 关键词打分模式必须上传，AI 模式可不上传
- 建议上传英文简历，关键词匹配更准确

## 三、职位列表

配置页「职位列表」Tab 可查看所有匹配到的职位：

- **排序**：发现时间 / 匹配分数 / 公司名 / 发布时间，点击表头切换
- **筛选**：公司名搜索、状态筛选、日期范围、仅看收藏
- **每条职位操作**：
  - ⭐ 收藏（特别感兴趣的职位）
  - 状态标记（未处理 / 已投递 / 不感兴趣）
  - 添加备注
  - 🗑 移入垃圾箱
- **垃圾箱**：可恢复或永久删除
- **导出**：CSV（Excel）或 JSON，选择格式后点导出下载

## 四、日常使用流程

```
1. 双击 Double Click to Start_Mac.command（或 Windows.bat）打开配置页
2. 确认配置无误 → 点击「启动 Job Tracker」
3. 关闭浏览器，程序继续在后台运行
4. 有新匹配职位 → TG / Email 收到推送
5. 在「职位列表」Tab 管理和追踪投递进度
```

重启 Mac 后需要重新双击启动，程序不会开机自启。

## 五、常见问题

**Q：双击 .command 文件没反应 / 提示无法打开**
右键文件 → 打开 → 打开，macOS 安全策略限制，只需操作一次。

**Q：Windows 上双击 .bat 文件一闪而过**
右键 `.bat` 文件 → 以管理员身份运行，或在 cmd 中手动 cd 到程序目录后运行。

**Q：关键词别名学习要消耗多少 Token？**
学习一次（一个关键词一种语言）约消耗 200–400 个 Token，按 DeepSeek 计费约 $0.0001。缓存后不再重复调用，日常运行几乎零额外消耗。

**Q：提示"需要 Python 3.10 或更高版本"**
启动脚本会自动尝试通过 Homebrew 安装。若没有 Homebrew，终端会打印两种安装方式（Homebrew / 官网下载），按提示操作后重新双击启动即可。

**Q：浏览器没有自动打开**
启动脚本会等待服务就绪后再自动打开浏览器。若等待超时（终端提示 ❌），说明服务启动失败，请查看终端中的报错信息定位原因。也可手动访问 `http://localhost:8080`。

**Q：依赖安装失败**
终端会显示具体错误。检查网络后重新双击启动；若持续失败，可按终端提示手动运行 pip 安装命令。

**Q：收不到 TG 推送**
检查 Bot Token 和 Chat ID 是否正确，确认先给 Bot 发过一条消息。

**Q：提示 LinkedIn 限制 / 抓取失败**
抓取间隔调高到60分钟以上，程序会自动暂停一段时间后恢复。

**Q：关键词模式没有匹配到职位**
降低最低分数阈值，或检查简历文件是否上传成功，建议用 .txt 格式。

## 六、附录：Google Sheets 配置步骤

1. 进入 [Google Cloud Console](https://console.cloud.google.com)
2. 创建项目 → 启用 Google Sheets API
3. 创建 Service Account → 下载 JSON 凭证文件
4. 将 JSON 文件改名为 `credentials.json` 放入 Job Tracker 文件夹
5. 创建一个 Google Sheets，将 Service Account 邮箱加为编辑者
6. 复制 Sheets URL 中的 ID（`/d/` 和 `/edit` 之间的部分）填入配置
